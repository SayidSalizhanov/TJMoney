# TJMoney — Система управления личными финансами

TJMoney — это полнофункциональное веб-приложение для личного и группового управления финансами. Позволяет отслеживать транзакции, устанавливать финансовые цели, работать в группах, общаться в чате в реальном времени и получать ML-прогнозы будущих расходов.

---

## Возможности

- **Учёт транзакций** — добавление, редактирование, удаление транзакций доходов/расходов по категориям; массовый импорт через CSV
- **Финансовые цели** — создание и отслеживание прогресса личных или групповых целей
- **Групповая работа** — создание групп, управление участниками, совместные транзакции и цели
- **Чат в реальном времени** — групповой чат на WebSocket с историей сообщений
- **ML-прогноз расходов** — модель Random Forest предсказывает расходы по категориям на следующий месяц
- **Курсы валют** — актуальные обменные курсы через FX Rates API
- **Напоминания** — запланированные уведомления с отправкой по email
- **Обучающие статьи** — финансовые статьи, публикуемые администраторами
- **Управление аватаром** — загрузка фото профиля через Cloudinary

## Доменная модель

| Сущность | Описание |
|----------|----------|
| `User` | Пользователь приложения с ролями, аватаром, Telegram ID |
| `Group` | Группа финансового взаимодействия |
| `GroupMember` | Запись об участии пользователя в группе с ролью (ADMIN / MEMBER) |
| `Transaction` | Запись дохода или расхода с категорией, суммой, датой |
| `Goal` | Финансовая цель с отслеживанием прогресса |
| `Record` | Текстовая заметка или документ внутри группы |
| `Reminder` | Запланированное уведомление, отправляемое по email |
| `Article` | Обучающая финансовая статья (создаётся администратором) |
| `ChatMessageEntity` | Сохранённое сообщение чата в реальном времени |
| `Application` | Заявка на вступление в группу (ожидает / принята / отклонена) |
| `Avatar` | Фото профиля, хранимое в Cloudinary |
| `Role` | Роль пользователя (USER / ADMIN) |

### Ключевые связи
- User ↔ Group: Many-to-Many через `GroupMember`
- User → Transactions, Goals, Records, Reminders: One-to-Many
- Group → Transactions, Goals, Records, Chat Messages: One-to-Many
- User → Avatar: One-to-One
- User ↔ Role: Many-to-Many

### Категории транзакций
`Food` (Еда), `Transport` (Транспорт), `Housing` (Жильё), `Entertainment` (Развлечения), `Clothing` (Одежда), `Health` (Здоровье), `Education` (Образование), `Other` (Прочее)

## ML-прогноз расходов

ML-сервис предоставляет эндпоинт прогнозирования, вызываемый REST API по пути `/api/transactions/predict-expenses`.

- **Модель**: Random Forest Regressor, обучается для каждого пользователя и категории отдельно
- **Входные данные**: исторические транзакции, сгруппированные по месяцу и категории
- **Выходные данные**: прогнозируемые суммы расходов по категориям на следующий месяц
- **Граничные случаи**: корректно обрабатывает недостаточный объём данных; обрезает выбросы на уровне 95-го процентиля

## Безопасность

- JWT Bearer токены со сроком действия 24 часа
- Хэширование паролей BCrypt
- Ролевой контроль доступа: `USER` и `ADMIN`
- Управление сессиями без сохранения состояния (stateless)
- Публичные эндпоинты: `/api/auth/**`, `/api/articles` (чтение), `/api/currency/**`, WebSocket

## Запуск проекта

### Требования
- Установленные Docker и Docker Compose

### Запуск всех сервисов

```bash
docker-compose up --build
```

| Сервис | URL |
|--------|-----|
| Фронтенд | http://localhost:3000 |
| REST API | http://localhost:8080 |
| ML Сервис | http://localhost:8000 |
| Swagger UI | http://localhost:8080/swagger-ui.html |

### Переменные окружения / Конфигурация

Основные настройки находятся в `rest-api/impl/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/tj-money
jwt:
  expiration: 86400000   # 24 часа
cloudinary:
  # Учётные данные Cloudinary API
fxRatesApi:
  # Ключ FX Rates API
mail:
  # Учётные данные SMTP
```
